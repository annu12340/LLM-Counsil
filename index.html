§ 02 — The Case For It

Why not just ask ChatGPT?

Because one model only ever argues with itself. It plays every side at once, so it hedges — lists the pros and cons, leans gently one way, and never gets challenged on the flaw in its favorite answer. The Council is built on disagreement: five experts argue against each other, so the weak option gets attacked before you build on it.

Asking one AI model

One opinion, hedged

  • Argues with itself, then sits on the fence
  • Nobody challenges the weak option
  • Ends on "it depends…"
  • No record of why it leaned that way
Convening the council

Five experts, one verdict

  • Five experts argue opposing sides
  • Every assumption gets cross-examined
  • A judge commits to one recommendation

§ 04 — How It Works

How the council reaches a verdict

Every question follows the same path — so the recommendation is something you can trust and defend, not a coin flip.

I

Submit the case

Ask an ADR/RFC question or repo-grounded technical decision.

II

Experts take sides

Five AI experts open from different angles.

III

They cross-examine

Each one attacks the others' weak points.

IV

Positions update

Strong arguments survive; shaky ones fold.

V

The judge rules

One recommendation, with confidence + next steps.

VI

The verdict is filed

The full debate, saved as shareable HTML.

§ 07 — Architecture

How the skill is built

LLM Council is a procedure-driven Agent Skill — not a server. The agent loads instructions at session start, runs a fixed debate pipeline in-chat, and writes a self-contained HTML artifact to disk. A bundled local tool now proves that artifact contract end to end.
